# bridge-preamble — operator style for BRIDGED (human-in-the-loop) sessions (OPTIONAL)
#
# incoming-transmission is an agnostic transport: by default a bridged session
# behaves like normal collaborative Claude. You are in the topic, so there IS a
# human to ask — the transport adds no autonomy opinion. This file is the seam
# for layering your OWN operator style onto a bridged session.
#
# If a non-comment, non-empty version of this file exists at
# ~/.telegram-bridge/bridge-preamble.txt, poll-render.sh prepends its contents to
# the rendered poll prompt so each poll tick carries your guidance. With no such
# file (the default), nothing is injected and the bridged session behaves like
# vanilla collaborative Claude plus the transport mechanics.
#
# This applies to BRIDGED sessions (you attached via /telegram and are typing in
# the topic). For UNATTENDED `/new` spawns, use spawn-preamble.example.txt.
#
# Lines beginning with '#' and blank lines are ignored. To activate, copy this to
# ~/.telegram-bridge/bridge-preamble.txt and write your guidance below (without
# the '#').
#
